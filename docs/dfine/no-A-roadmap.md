# 无 A 路线计划

## 背景

当前用户已明确：
- 不希望继续使用 Method A。
- 原因是无法收集完备负样本，且原有 A 路线会污染 OA 测试参考性。
- OA 测试集不应再进入训练。

## 当前路线原则

1. **OA 测试集独立**
   - 不进入训练
   - 只用于选模/盲测

2. **优先解决模型过于自信的问题**
   - 不是继续堆外部 negative images

3. **如果需要训练负监督，来源应更可控**
   - 从现有带球图像中，在非排球区域随机裁切出局部负样本
   - 或在训练机制上抑制高置信误激活

## 分阶段计划

### Phase 1：可信评估体系
先不急着改训练，先保证评估可信：
- 划分独立 OA 选模集
- 划分最终 blind OA 测试集
- 统一 OA 评估协议

### Phase 2：平台链路验证
- 先通过 GradMotion smoke test
- 再做极短训练闭环
- 确认云平台、Git checkout、数据路径与日志链路都可靠

### Phase 3：验证当前分支
- 当前 git 分支方案还未测试
- 在不使用 A 的前提下，先验证当前分支真实表现
- 比较项重点看：
  - AP
  - OA clean rate
  - OA max false score
  - 部署阈值下的误检表现

### Phase 4：再决定是否引入更强抑制手段
候选方向：
- 更合理的 background/self-confidence 抑制
- 正图内部随机非球裁切监督
- query 数量与匹配策略微调
- 推理后处理的轻量过滤

## 当前不建议直接做的事

1. 不建议继续把 `negative_samples.zip` 合进训练集。
2. 不建议直接长时间开训验证 E2，先过 smoke test。
3. 不建议把 `bg_loss_weight=6` 当成默认真理，先验证当前分支与评估体系。

## 当前推荐动作顺序

1. 完成 GradMotion smoke test
2. 完成极短训练闭环
3. 整理独立 OA 评估集
4. 测试当前分支
5. 再决定是否需要新的 no-A 训练设计
